# Channel Discovery Run — 2026-06-01

Goal: surface **new dofollow-traffic channels** not yet integrated, ranked by
`indexed-dofollow value × low adapter cost × not-a-bad-neighborhood`, deduped
against everything already decided.

## Method (what the probe actually did)

1. **Dedup** against the decided-store: 18 registered platforms + retired
   (`bloglovin`, removed CN `jianshu/csdn/juejin/note`) + HOLD (`jkforum`) +
   deferred (`justpaste.it`, `teletype.in`). None re-probed.
2. **HTTP-UA matrix** — `scripts/channel_probe.py` per candidate (preflight-bot /
   Googlebot / browser UA) → reachability / login-wall / Cloudflare verdict.
3. **dofollow tier** — fetched a **real public post** on each reachable candidate
   and ran the project's own `link_attr_verifier.verify_link_attributes()` to read
   the actual `rel` on outbound external links (3rd-party live check — *not* an
   OUR-pipeline canary; see Honesty note).
4. **indexed / mechanism / cost / bad-neighborhood** — WebSearch + curl evidence.

> ⚠️ **SSRF caveat (funnel R14).** `channel_probe.py` does **not** route through
> `net_safety`. This run probed only **hand-curated public hostnames**, so the risk
> is bounded — but the probe must adopt the production SSRF guard before it is ever
> driven on machine-sourced URLs.

## Per-candidate evidence (GO tier)

**1. GitLab Pages — GO-reuse (highest ROI).** `gitlab.com` → 200 to all UAs, no
CF/login. Pages output is the operator's own static HTML on `*.gitlab.io`, so `rel`
is operator-controlled and dofollow by construction — identical trust model to our
`ghpages` adapter. Free account + email verify; public Pages free. **Generalize the
`GitHubPagesAPIAdapter` (parameterize git host / API base / published-URL) → a
`gitlabpages` channel.** Skip GitLab **Snippets** — long-standing indexation gap
(GitLab issues #4650/#24289: GitLab-only content barely appears in Google).

**2. HackMD — GO-new-light (dofollow CONFIRMED).** `verify_link_attributes` on a real
note (`hackmd.io/UyWGSZQBQ4-lde2PaoBS1g`) → **188 anchors, 0 nofollow, index,follow**
(re-verified directly, not just by sub-agent). Clean Bearer-token REST:
`POST https://api.hackmd.io/v1/notes` with `readPermission=guest`. DA~71. Maps almost
drop-in onto the **devto/notion token-REST archetype**.

**3. Mataroa — GO-new-light (dofollow CONFIRMED).** `verify_link_attributes` on
`mataroa.blog/blog/` → **4 anchors / 0 nofollow** and a real post → **6 / 0**
(re-verified directly). Open REST API: `POST https://mataroa.blog/api/posts/`,
`Authorization: Bearer <api-key>`. `site:mataroa.blog` shows fresh indexed content.
Near-identical to the devto archetype. (Watch for future nofollow/noindex policy — it
currently tolerates marketing posts, which means it *could* tighten.)

**4. Read the Docs — GO-new-light (medium cost).** Real docs page (`requests.readthedocs.io`)
→ 0/131 nofollow, sitemap + index,follow. Free community tier (ads). Cost is **medium**:
needs a buildable Sphinx/MkDocs repo, not a single API PUT — and RTD's abuse tolerance
is lower, so pair with *genuine* docs content, not thin SEO pages.

**5. notes.io — GO-new-light (diversification only).** Real note (`notes.io/HFtz`) →
12 anchors / 0 nofollow, server-rendered, index,follow. **Anonymous** form-post (no
account) → reuses the **txtfyi** archetype. But: same paste-style archetype we already
have ×3 (rentry/txtfyi/telegraph) and high footprint/bad-neighborhood risk → add only
as low-priority diversification behind an indexation canary.

**6–7. Qiita + Zenn — NEEDS-CANARY (nofollow, high referral).** Both inject
`rel="nofollow noopener[ noreferrer]"` on every outbound external link (verified on
real articles: Qiita 12/86, Zenn 36/137). **Zero PageRank transfer** — they pass the
value gate *only* on referral value, which is high (top-tier JP dev domains). Qiita =
token REST v2 (devto reuse, cheapest); Zenn = GitHub-repo git-push (ghpages reuse).
Register as `dofollow=False, referral_value="high"` (the linkedin/notion/devto pattern),
**not** as dofollow channels. JP-language audience gate applies.

## Honesty / scope notes

- **"GO" ≠ delivered SEO equity.** dofollow was confirmed on *other users'* posts
  (3rd-party live check). "Converts to a live channel" = registered + publishable +
  **OUR**-post dofollow-canary-confirmed — not that our link is indexed and passing
  equity (the project has no indexation/GA4 loop to prove that). Per-link indexation
  verification stays a named follow-on.
- **Two `git-push`/`token-REST` archetypes carry the round.** The funnel brainstorm's
  WriteFreely-family thesis was killed by per-instance-account cost. HackMD + Mataroa
  sidestep that entirely: **free public posting + dofollow via a clean token API**, no
  per-instance account farming. That is a strictly better answer than family-enum.
- **Nothing was registered, nothing was published.** This is a read-only report.
