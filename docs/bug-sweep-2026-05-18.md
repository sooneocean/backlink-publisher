### L5 Orchestration (1 fix)

The single failure was a test-side regression caused by drift between the test's hardcoded seed domain and the operator's local config. The test (`test_plan_no_synthesized_categories_url_without_config`) is a regression guard against `_build_links` synthesizing hardcoded `/categories` and `/detail` URLs without `[sites.*].url_categories` config support. It was written when `main_domain=https://51acgs.com/` always routed through `_build_links`. After the work-themed dispatcher landed (`_plan_work_themed_row` triggered by `[targets."<domain>"]` config), the same seed silently routed to a different code path, which then fell through `work_scraper.fetch_work_metadata` (not autouse-mocked) under disabled-socket → zero payloads → empty stdout → JSON decode error.

The fix is one line of semantic intent: choose a domain not in the operator's `[targets.*]` table. `example.com` satisfies both the test's "no config" precondition and the routing precondition. The test's invariants (no synthesized `/categories`, no `/detail`) are unchanged.

**Test-modification category**: `contract-evolution`. The source legitimately evolved (work-themed dispatch is intentional behavior); the test's seed domain choice was implicit on a since-changed assumption.

## Follow-ups

1. **Test environment hardening (deferred)**: This bug class repeats — tests that depend on `~/.config/backlink-publisher/config.toml` are intrinsically fragile against operator state. Consider:
   - Adding a session-scope conftest fixture that points `XDG_CONFIG_HOME` or `--config` at an isolated temp config dir (deterministic, operator-state-independent).
   - Or grepping `tests/` for hardcoded real-target-domain strings (`51acgs.com`, `taiwanmanga2026.blogspot.com`) and migrating them to `example.com` / `example.org` / `example.net`.

2. **Conftest string-match special case** (`tests/conftest.py:63` `"test_content_fetch.py" in test_path`): structural hazard — silently re-engages if file is renamed. Consider migrating to a pytest marker (parallel to `real_ssrf_check`). Out of scope here; logged as deferred per plan.

3. **Plan over-scoping**: The plan was sized for "53 test files, unknown failures, multi-hour sweep". Reality was 1 failure × 1 file × 12s wall time. The brainstorm's `if 数量 > 30, 缩范围` hedge correctly anticipated this but the plan still spent significant ceremony on layered units, hotspot tables, dual stop-gates, and contract-evolution category gates that turned out to be unused. Take-away: **run a 12-second `pytest` BEFORE writing a multi-section plan** when the problem space is "sweep" rather than "build". A 30-line plan keyed off "1 failure, here's the fix" would have shipped faster.

4. **PR coordination**: No surfaces touched by this sweep overlap with active PRs #36 (docs only), #38 (docs/spike only), or #39 (webui — `tests/test_webui_*` are untouched by this fix). PR #37 was already merged before sweep started.
